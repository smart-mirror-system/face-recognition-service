# Mrayti AI Biometrics Service - API Documentation

## Overview
This document serves as the official API reference for the **Mrayti AI Biometrics Service**. This is a high-performance, GPU-optimized Python/FastAPI microservice dedicated strictly to FaceNet512 facial recognition, Qdrant vector storage, and biometric authentication.

*Note for the Admin Dashboard and Node.js Teams:* To maintain strict architectural integrity, external services and the Admin Dashboard must interact exclusively with these documented endpoints and never attempt direct connections to the Qdrant database.

---

## 1. Authentication & Authorization

All secure endpoints expect a JSON Web Token (JWT) provided in the `Authorization` header. This token is originally generated by the core Node.js service.

**Header Format:**
`Authorization: Bearer <your_jwt_token>`

The AI service will decode this token using the shared `JWT_SECRET`. The JWT payload MUST contain the unique user identifier in the `userId` claim.

---

## 2. API Endpoints

This service is routed through the API Gateway under the `/api/v1/ai` prefix. The gateway strips this prefix before forwarding the request to the internal Python container.

### 2.1 Register Face Pose
Registers a facial embedding for the currently authenticated user. This should be called multiple times during the initial user onboarding flow to capture various angles.

* **Gateway URL:** `/api/v1/ai/register`
* **Internal Container Path:** `/register`
* **Method:** `POST`
* **Content-Type:** `multipart/form-data`
* **Rate Limit:** 10 requests per minute

**Headers:**
* `Authorization: Bearer <token>` (Required)
* `Content-Length: <bytes>` (Required for size validation, max 5MB)

**Request Body (Form Data):**
* `file`: The captured webcam image (`.jpg`, `.jpeg`, `.png`)

**Success Response (201 Created):**
```json
{
  "success": true,
  "message": "Face vector successfully registered.",
  "point_id": "b8a9c2... (UUID)",
  "user_id": "user_123"
}
```

**Error Responses:**

- `400 Bad Request`: Invalid file format.
- `401 Unauthorized`: Missing, expired, or invalid JWT.
- `422 Unprocessable Entity`: No face detected in the frame.

### 2.2 Face Login (Biometric Auth)
Authenticates a user standing in front of the mirror by comparing their live webcam frame against the stored vector database (Qdrant).

* **Gateway URL:** `/api/v1/ai/login`
* **Internal Container Path:** `/login`
* **Method:** `POST`
* **Content-Type:** `multipart/form-data`
* **Rate Limit:** 5 requests per minute

**Request Body (Form Data):**
* `file`: The live webcam frame (`.jpg`, `.jpeg`, `.png`)

**Success Response (200 OK):**
On success the service issues a signed JWT identical to what the Node.js API would generate. This token can be used immediately for authenticated requests (`/register`, `/faces/{point_id}`, etc.).

```json
{
  "ok": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "userId": "6a3b1a6a75b4fd165ed0da44"
}
```

The token is signed with the shared `JWT_SECRET` and expires after `JWT_EXPIRES` (default `7d`).

**Error Responses:**

- `400 Bad Request`: Invalid file format.
- `401 Unauthorized`: Face matched, but confidence score was below the secure threshold of 0.65 (Stranger detected).
- `422 Unprocessable Entity`: No face detected in the frame.

### 2.3 Delete Face Point
Removes a single registered face vector by its unique point ID. The authenticated user can only delete their own face vectors.

* **Gateway URL:** `/api/v1/ai/faces/{point_id}`
* **Internal Container Path:** `/faces/{point_id}`
* **Method:** `DELETE`
* **Rate Limit:** 30 requests per minute

**Headers:**
* `Authorization: Bearer <token>` (Required)

**Success Response (200 OK):**
```json
{
  "success": true,
  "message": "Face vector successfully deleted.",
  "point_id": "b8a9c2... (UUID)"
}
```

**Error Responses:**

- `401 Unauthorized`: Missing, expired, or invalid JWT.
- `403 Forbidden`: The point belongs to a different user.
- `404 Not Found`: No face vector found with the given point ID.

### 2.4 Delete All User Faces
Removes all registered face vectors belonging to the authenticated user. Used when a user account is deleted or the user wants to reset their face data.

* **Gateway URL:** `/api/v1/ai/faces/user/{user_id}`
* **Internal Container Path:** `/faces/user/{user_id}`
* **Method:** `DELETE`
* **Rate Limit:** 10 requests per minute

**Headers:**
* `Authorization: Bearer <token>` (Required)

**Success Response (200 OK):**
```json
{
  "success": true,
  "message": "Successfully deleted 3 face vector(s).",
  "deleted_count": 3
}
```

**Error Responses:**

- `401 Unauthorized`: Missing, expired, or invalid JWT.
- `403 Forbidden`: The target `user_id` does not match the authenticated user.
- `404 Not Found`: No face vectors found for this user.

### 2.5 Health & Readiness Probes
Used by Docker and the API Gateway to monitor container status and orchestrate zero-downtime deployments.

#### Liveness Probe
Indicates if the container process is running.

- **URL:** `/health`
- **Method:** `GET`
- **Success Response (200 OK):**
```json
{
  "status": "alive"
}
```

#### Readiness Probe
Verifies if downstream dependencies (Qdrant and Redis) are fully operational and ready to accept traffic.

- **URL:** `/ready`
- **Method:** `GET`
- **Success Response (200 OK):**
```json
{
  "status": "ready",
  "qdrant": "connected",
  "redis": "connected"
}
```

- **Error Response (503 Service Unavailable):**
```json
{
  "status": "unready",
  "dependencies": {
    "qdrant": "Unreachable or initializing",
    "redis": "Ping failed"
  }
}
```
